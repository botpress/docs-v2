## Configuration File

The application looks for a `config.json` file in the root of your project. Here's a complete example:

```json
{
  "name": "my-app",
  "version": "1.0.0",
  "port": 3000,
  "database": {
    "host": "localhost",
    "port": 5432,
    "name": "mydb"
  },
  "features": {
    "analytics": true,
    "notifications": true,
    "darkMode": false
  }
}
```

## Environment Variables

You can also configure the application using environment variables. These take precedence over the config file:

| Variable    | Description       | Default     |
| ----------- | ----------------- | ----------- |
| `APP_PORT`  | Server port       | `3000`      |
| `APP_DEBUG` | Enable debug mode | `false`     |
| `DB_HOST`   | Database host     | `localhost` |
| `DB_PORT`   | Database port     | `5432`      |
| `DB_NAME`   | Database name     | —           |

## TypeScript Configuration

If you're using TypeScript, add the following to your `tsconfig.json`:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "strict": true,
    "esModuleInterop": true
  }
}
```

<Tip>Use `strict: true` to catch potential issues early. It enables all strict type-checking options.</Tip>

## Feature Flags

Feature flags allow you to enable or disable functionality at runtime:

```typescript
import { features } from 'my-package'

if (features.isEnabled('analytics')) {
  trackPageView()
}
```

### Available Flags

- **analytics** — Enable usage analytics
- **notifications** — Enable push notifications
- **darkMode** — Enable dark mode by default
- **experimentalApi** — Enable experimental API endpoints

<Danger>Experimental features may change or be removed without notice. Do not rely on them in production.</Danger>
