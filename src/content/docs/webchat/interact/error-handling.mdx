When Webchat encounters an error, it displays a dedicated error screen instead of the normal chat interface. Here's a breakdown of all possible error screens:

## Configuration Errors

### Missing client ID

<Frame>
  <Picture alt="Missing client ID" class="block dark:hidden" src={missingClientIdImg} />
  <Picture alt="Missing client ID" class="hidden dark:block" src={missingClientIdDarkImg} />
</Frame>

- **Cause**: Client ID not provided during initialization
- **Solution**: Provide a valid client ID

### Invalid client ID

<Frame>
  <Picture alt="Invalid client ID" class="block dark:hidden" src={invalidClientIdImg} />
  <Picture alt="Invalid client ID" class="hidden dark:block" src={invalidClientIdDarkImg} />
</Frame>

- **Cause**: Provided Client ID isn't valid
- **Solution**: Use a valid Botpress Client ID

### Missing API URL

<Frame>

## Bot Errors

### Bot is blocked

<Frame>
  <Picture alt="Bot is blocked" class="block dark:hidden" src={botIsBlockedImg} />
  <Picture alt="Bot is blocked" class="hidden dark:block" src={botIsBlockedDarkImg} />
</Frame>

- **Cause**: Bot has been temporarily paused
- **Solution**: Check your bot's [status](/get-started/manage-your-agent/inspect)

### Quota exceeded

<Frame>
  <Picture alt="Quota exceeded" class="block dark:hidden" src={quotaExceededImg} />
  <Picture alt="Quota exceeded" class="hidden dark:block" src={quotaExceededDarkImg} />
</Frame>

- **Cause**: Bot has reached usage limits
- **Solution**: Check usage quotas and limits in [Dashboard](/get-started/configure-your-workspace#manage-usage-quotas)

### Not a participant

<Frame>
  <Picture alt="Not a participant" class="block dark:hidden" src={notAParticipantImg} />
  <Picture alt="Not a participant" class="hidden dark:block" src={notAParticipantDarkImg} />
</Frame>

- **Cause**: User not authorized for conversation
- **Solution**: Clear browser storage and refresh

### Invalid message content type

<Frame>
  <Picture alt="Invalid message content type" class="block dark:hidden" src={invalidMessageContentTypeImg} />
  <Picture alt="Invalid message content type" class="hidden dark:block" src={invalidMessageContentTypeDarkImg} />

- **Cause**: Unable to process message format
- **Solution**
  - Check your bot's logs for any integration errors
